# Testing Rules

- Minimalny baseline przed zakończeniem zmian: `npm run lint` i `npm run build`.
- Dla zmian UI sprawdź render desktop + mobile przez `npm run dev`.
- Nie generuj pełnych logów build/test bez potrzeby; limituj output do istotnych fragmentów.
- Jeżeli pełny log jest potrzebny, zapisuj go do pliku i raportuj tylko kluczowe linie.
